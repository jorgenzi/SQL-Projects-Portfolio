В этом проекте освоены ключевые практические навыки SQL-анализа, необходимые для решения реальных бизнес-задач: агрегация данных (расчет сумм, средних, минимума/максимума), группировка по атрибутам, фильтрация результатов, соединение таблиц и работа с условиями.

Я научилась превращать сырые данные из базы в структурированные отчеты, рассчитывать метрики эффективности, анализировать поведение клиентов и принимать обоснованные бизнес-решения на основе данных.


Задание 00:
Выполнена простая агрегация. SQL-запрос, который возвращает идентификаторы людей (person_id) и соответствующее количество посещений любых пиццерий.

Результат отсортирован по количеству посещений в порядке убывания, а затем по идентификатору человека в порядке возрастания.

Задание 01:
Изменён SQL-запрос из Задания 00 так, что он возвращает имя человека (name), а не его идентификатор.

Дополнительное условие - выводится только топ-4 человека с максимальным количеством визитов в каждой пиццерии, отсортированных по имени человека.

Задание 02:
SQL-запрос, который выводит топ-3 заведения, которые являются самыми популярными по количеству посещений и по количеству заказов, результаты объединены в один список.

Добавлен столбец action_type, который содержит значения 'order' или 'visit' в зависимости от того, из какой таблицы взяты данные.
Итоговый результат отсортирован в порядке возрастания по столбцу action_type и в порядке убывания по столбцу с количеством (count).

Задание 03:
SQL-запрос, который показывает, как рестораны группируются по количеству посещений и по количеству заказов и объединяет эти данные по названию пиццерии.

Добавлены следующие правила:
- Вычислена общая сумма заказов и посещений для каждой пиццерии
- Результаты отсортированы по столбцу total_count в порядке убывания и по столбцу name в порядке возрастания

Задание 04:
SQL-запрос, который возвращает имя человека (person name) и соответствующее количество посещений любых пиццерий при условии, что это количество превышает 3 раза (> 3).

Задание 05:
SQL-запрос, который возвращает список уникальных имен людей, сделавших хотя бы один заказ в любой из пиццерий.
Результат отсортирован по имени человека.

Задание 06:
SQL-запрос, который возвращает для каждой пиццерии:

- общее количество заказов,
- среднюю цену,
- максимальную цену
- и минимальную цену на проданные пиццы.

Результат отсортирован по названию пиццерии. Средняя цена округлена до двух знаков после запятой.

Задание 07:
SQL-запрос, который возвращает общий средний рейтинг (выходной атрибут - global_rating) для всех ресторанов.
Среднее значение рейтинга округлено до 4 знаков после запятой.

Задание 08:
SQL-запрос, который возвращает:

1. Адрес (person address)
2. Название пиццерии (pizzeria name)
3. Число заказов (count of orders) в этой пиццерии.

Результат отсортирован сначала по адресу, а затем по названию заведения.

Задание 09:
SQL-запрос, который возвращает агрегированную информацию по адресу каждого человека.

В результат включены:

- вычисляемый столбец с формулой: «Максимальный возраст - (Минимальный возраст / Максимальный возраст)»,
- средний возраст по каждому адресу (average age per address),
- результат сравнения формулы и среднего возраста.

Результат отсортирован по столбцу с адресом.
