В этом проекте я освоила продвинутые техники работы с индексами PostgreSQL: научилась создавать разные типы индексов, анализировать запросы и оптимизировать их так, чтобы все работало максимально быстро.

Задание 00:                                                                 
Создан простой BTree index для каждого внешнего ключа в нашей базе данных.
Шаблон имени соответствует следующему правилу: idx_{имя_таблицы}_{имя_столбца}.

Например, имя индекса типа B-дерево для столбца pizzeria_id в таблице `menu` будет `idx_menu_pizzeria_id`.

Задание 01: 
SQL-запрос, который возвращает названия пицц и соответствующие названия пиццерий.
Убеждаемся, что индексы работают для моего SQL-запроса. 
В качестве доказательства потребуется вывод команды `EXPLAIN ANALYZE`.

Задание 02:

SQL-запрос с доказательством работы индекса (`EXPLAIN ANALYZE`), демонстрирующий, что индекс idx_person_name используется.
