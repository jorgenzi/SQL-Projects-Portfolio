В этом проекте я научилась реализовывать бизнес-требования, обеспечивать целостность данных через ограничения и индексы, оптимизировать производительность БД, работать с последовательностями и массовыми операциями, а также документировать структуру базы.

Задание 00:
Добавлена новая бизнес-возможность в модель данных.
Реализована система персональных скидок для клиентов с одной стороны и пиццерий — с другой. Создана новая таблица отношений person_discounts со следующими правилами:

- Определён атрибут id в качестве Primary Key.
- Определены атрибуты person_id и pizzeria_id как Foreign Keys для соответствующих таблиц.
- Заданы явные имена для ограничений внешнего ключа.
- Добавлен атрибут discount для хранения значения скидки в процентах.

Задание 01:
Заполнена таблица person_discounts новыми записями.
Написан DML-оператор (INSERT INTO ... SELECT ...), который заполнил таблицу person_discounts новыми записями, следуя приведенным правилам:

- Сгруппированы данные по столбцам person_id и pizzeria_id.
- Рассчитан размер персональной скидки по следующему псевдокоду:
  - Если количество заказов = 1, то скидка = 10.5
  - Иначе если количество заказов = 2, то скидка = 22
  - Иначе скидка = 30

Задание 02:
SQL-запрос, который возвращает список заказов с фактической стоимостью и стоимостью с примененной скидкой для каждого клиента в соответствующей пиццерии. Отсортированы результаты по имени клиента и названию пиццы.

Задание 03:
Создан уникальный многоколоночный индекс (с именем idx_person_discounts_unique), который предотвращает дублирование пар идентификаторов person_id и pizzeria_id.

Задание 04:
Добавлены следующие правила ограничений для столбцов таблицы person_discounts:

1. Столбец person_id не содержит NULL.
2. Столбец pizzeria_id не содержит NULL.
3. Столбец discount не содержит NULL.
4. Столбец discount имеет значение по умолчанию 0 (0%).
5. Столбец discount содержит значения в диапазоне от 0 до 100.

Задание 05:
В соответствии с политикой управления данными (Data Governance Policies), добавлены комментарии к таблице и ее столбцам, объясняющие бизнес-назначение таблицы и всех ее атрибутов.

Задание 06:
Cоздана последовательность базы данных с именем seq_person_discounts (начиная со значения 1) и установлено значение по умолчанию для атрибута id таблицы person_discounts, чтобы оно автоматически бралось из seq_person_discounts при каждой вставке.
