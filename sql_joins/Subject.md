В этом проекте я освоила основные навыки работы с SQL и реляционными базами данных: понимание реляционной алгебры, использование различных типов соединений (JOIN), создание запросов с помощью CTE, обработку пропущенных значений, фильтрацию и сортировку данных.

Задание 00:
SQL-запрос, который выводит список пиццерий с их рейтингом, но только те, которые никто не посещал.

Задание 01:
SQL-запрос, который возвращает отсутствующие даты с 1 по 10 января 2022 года (включительно) для посещений людьми с идентификаторами 1 и 2 — то есть дни, пропущенные обоими. Результат отсортирован по дате посещения в порядке возрастания.

Задание 02: 
SQL-запрос, который возвращает:
- Полный список имен людей, посетивших (или не посетивших) пиццерии в период с 1 по 3 января 2022 года
- Полный список названий пиццерий, которые посещались (или не посещались) в этот же период
Сортировка по всем трем колонкам.

Задание 03:
SQL-запрос, используя CTE (Common Table Expression). В CTE-блоке реализован генератор дат. Результат должен совпадать с результатом из Задания 01.

Задание 04:
Запрос для поиска полной информацию обо всех возможных вариантах пиццерий и ценах на пиццу с грибами или пепперони.
Результаты отсортированы по названию пиццы и названию пиццерии.

Задание 05:
Найдены имена всех женщин старше 25 лет и результат отсортирован по имени. 

Задание 06:
Найдены все названия пицц (и соответствующие названия пиццерий из таблицы menu), которые заказывали Денис или Анна.
Результат отсортирован по обеим колонкам.

Задание 07: 
Запрос для определения названия пиццерии, которую Дмитрий посетил 8 января 2022 года, где он мог заказать пиццу дешевле 800 рублей.

Задание 08: 
Найдены имена всех мужчин из Москвы или Самары, которые заказывали пиццу с пепперони, грибами или оба вида сразу.
Результат отсортирован по именам в обратном алфавитном порядке.

Задание 09: 
Найдены имена всех женщин, которые заказывали и пепперони, и сырную пиццу (в любое время и в любых пиццериях).
Результат отсортирован по именам в алфавитном порядке.

Задание 10:
Найдены имена людей, проживающих по одному адресу (в одном городе).
Результат отсортирован в три колонки: по имени первого человека, имени второго человека и общему адресу.
